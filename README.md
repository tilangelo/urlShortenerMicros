# URL-SHORTENER на микросервисах
## Состав:
### 1. core(8080)
* Создание короткой ссылки(shortUrl)
* Генерация id методом snowflake
* ShortUrl состоит из shortCode - генерируется Base62 из id
* MVC сервис, работает с pg, redis.
### 2. redirect(8081)
* Реактивный сервис для перехода на ресурс за shortUrl
* Общается только с reactive redis, без фоллбеков на блокирующий сервис
### 3. api-gateway(8082)
* Общий прокси узел
* Валидация(при необходимости) ip, auth, времени и пр
* Метрики

# Endpoints API Gateway

## Обзор
API Gateway работает на порту `8082` и направляет запросы к соответствующим микросервисам на основе URL-паттернов.

## Основные Endpoints

### 1. Создание короткой ссылки
**Endpoint:** `POST /core-api/links`  
**Описание:** Создаёт новую короткую ссылку  
**Проксирует в:** Core Service (localhost:8080)
* Пример: {
  "longUrl": "https://github.com/tilangelo",
  "ttl": "3000000"
  }

### 2. Создание политики ссылки
**Endpoint:** `POST /core-api/policies`  
**Описание:** Создаёт политику доступа для короткой ссылки  
**Проксирует в:** Core Service (localhost:8080)
* Пример: {
"linkId": 299826759354814464,
"shortcode": "M9D0EcduHQ",
"allowedIps": ["105.55.15.31"],
"allowedTimeStart": "2026-04-07T08:00:00Z",
"allowedTimeEnd": "2026-04-07T15:00:00Z",
"authType": "NONE",
"authConfig": ""
}

### 3. Переход по короткой ссылке
**Endpoint:** `GET /{shortcode}`  
**Описание:** Редирект на оригинальный URL после валидации политики(при её присутствии)  
**Прямой маршрут:** Redirect Service (localhost:8081)  
**Процесс:** 
- Валидация политики через LinkPolicyFilter
- Проверка IP whitelist
- Валидация временного окна
- Валидация аутентификации (если требуется)
- Сбор метрик

### 4. Удаление политики ссылки
**Endpoint:** `DELETE /core-api/policies/{policyId}`  
**Описание:** Удаляет политику доступа для короткой ссылки  
**Проксирует в:** Core Service (localhost:8080)

## Дополнительные Endpoints

### Проверка здоровья
**Endpoint:** `GET /health/detailed`  
**Описание:** Детальный статус здоровья с метриками circuit breaker  
**Ответ:** JSON со статусом и метриками circuit breaker

### Internal API
**Endpoint:** `/internal/**`  
**Описание:** Внутренние endpoints сервиса(fallbacks и прочие)  
**Проксирует в:** Core Service (localhost:8080)  


## Безопасность

### Link Policy Filter
- **Валидация временного окна:** Проверка ограничений по времени доступа
- **IP Фильтрация:** Поддержка whitelist
- **Аутентификация:** Поддержка нескольких типов аутентификации
- **Circuit Breaker:** Предотвращение каскадных отказов
- **Метрики:** Трекинг доступа и мониторинг производительности

### Rate Limiting & Resilience
- **Повторы:** 2 попытки с экспоненциальным backoff
- **Circuit Breaker:** Открывается при высоком уровне ошибок
- **Таймауты:** 8s таймаут фильтра, 5s таймаут подключения
- **Пул соединений:** Эластичный пул с 500 макс. соединений

## Мониторинг
- **Prometheus:** `/actuator/prometheus`

## Ответы с ошибками
Все ошибки возвращают JSON формат:
```json
{
  "error": "Not Found",
  "message": "Short link not found"
}
```

Распространённые HTTP статус-коды:
- `403 FORBIDDEN`: Ограничения по IP/временному окну
- `404 NOT FOUND`: Короткая ссылка или политика не найдены
- `503 SERVICE UNAVAILABLE`: Circuit breaker открыт
- `504 GATEWAY TIMEOUT`: Таймаут запроса
- `410 GONE`: shotUrl удалён
